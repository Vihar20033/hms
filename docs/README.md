## How to Use

1. Read [docs/advanced-features.md](advanced-features.md) to understand why each advanced feature exists.
2. Read [docs/system-design.md](system-design.md) for architecture and sequence flow understanding.
3. Read [docs/module-working-guide.md](module-working-guide.md) for per-module implementation and flow details.
4. Read [docs/api-endpoints-reference.md](api-endpoints-reference.md) for API contracts and authorization expectations.
5. Use the dependency and trade-off sections during architecture reviews.
6. Update before/after impact notes when shipping major module changes.
